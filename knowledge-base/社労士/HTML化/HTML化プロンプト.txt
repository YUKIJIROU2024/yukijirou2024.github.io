添付した文書（マニュアル）の内容を、「読むドキュメント」ではなく、
「情報密度が高く、かつ視覚的に理解できるHTMLアーティファクト」として出力してください。

# 目的
- 人が短時間で理解できるようにする
- 情報を構造化し、視覚的に整理する
- 必要に応じてインタラクションを使って理解を助ける

# 出力形式
- 単一の自己完結HTML（外部リソース禁止）
- HTML + CSS + 必要ならJavaScriptを含める
- そのままブラウザで開いて読める状態

# デザイン要件
- Anthropicが推奨するように、Markdownよりもモダンで美しいCSSを適用し、図解や強調箇所が目立つようにする。
- モダンで見やすいUI（カード、余白、色分け）
- セクションごとに明確に区切る
- 目次（ナビゲーション）を付ける：左側に固定の目次（サイドバー）を設置し、クリックで各セクションへジャンプできるようにする。
- レスポンシブ（スマホでも見やすい）

# コンテンツ構造
必ず以下を含める：
1. サマリー（最初に結論）
2. 全体像（図または構造説明）
3. 詳細セクション（機能別/論点別）
4. 重要ポイントの強調
5. 必要に応じて比較表

# 表現ルール
- 長文禁止：文章は短く分割
- 可能な限り視覚要素を優先（表・図・カード）
- 重要部分は色・アイコンで強調
- 箇条書きを多用
- コードはハイライト付きで表示

# 追加要素（可能なら）
- タブ切り替え
- 折りたたみセクション：各項目を折りたたみ可能なアコーディオン形式（<details>タグなど）にし、必要な情報だけを展開して読めるようにする。
- 簡単なインタラクション：検索バーや、上部に戻るボタンを設置して操作性を高める。

# 禁止事項
- Markdown形式での出力
- テキスト中心のだらだらした説明
- 外部CSS/CDN依存

# ゴール
「スクロールするだけで理解できるUI」にすること
