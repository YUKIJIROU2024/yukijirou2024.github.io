Using Claude Code: The unreasonable effectiveness of HTML
Anthropic / 2026-05-20 / 約5分
⚡ 結論ファースト

Markdownで読ませるより、HTMLで理解させる。

Claude Codeの出力は、単なる文章から「視覚的・操作的な成果物」へ進化できる。 HTMLは、仕様書・計画・レビュー・調査レポートを、短時間で読めるUIに変えるための器になる。

1目的：人間が短時間で把握できる成果物
5強み：密度・視認性・共有・操作・文脈活用
∞表現：表、図、SVG、CSS、JS、フォーム、エディタ

全体像：HTMLは「読めるUI」になる

記事の主張を1枚の構造にすると、以下のようになります。

AIの出力形式を変えると、理解の質が変わる

Markdownは軽くて編集しやすい。一方、HTMLは視覚化・構造化・操作・共有に強い。Claude Codeのように文脈を大量に読めるAIでは、HTMLの表現力がそのまま理解しやすさにつながる。

なぜHTMLを使うのか

記事で挙げられている主な理由を、実務上の意味に置き換えて整理します。

01 情報密度

1画面に多くの意味を載せられる

表、図、色、余白、階層で、長い文章を圧縮して伝えられる。

02 視認性

読む負担を減らせる

100行超のMarkdownより、セクション化されたHTMLのほうが追いやすい。

03 共有

リンクで見せやすい

HTMLはブラウザで開ける。添付ファイルより読まれる可能性が高い。

04 操作

調整・選択・比較ができる

スライダー、タブ、ボタン、コピー機能で、資料が小さな編集環境になる。

05 文脈

Claude Codeの強みと合う

コードベースや履歴を読んだうえで、図解資料に変換できる。

06 継続利用

仕様・検証の参照資料になる

計画、探索、実装、検証の各段階でHTMLを残せる。

MarkdownとHTMLの比較

Markdownが不要というより、用途によってHTMLが有利になるという話です。

観点MarkdownHTML実務での意味
書きやすさ強いやや重い短いメモやREADMEはMarkdownで十分。
情報密度限定的非常に強い表・図・色・配置で理解を早める。
長文の読みやすさ弱くなりやすい強いサイドバー、タブ、折りたたみで読む量を制御。
共有表示環境に左右されるブラウザで見やすい同僚が開いてすぐ理解できる。
操作性ほぼ静的JSで操作可能比較、絞り込み、コピー、簡易編集ができる。
AIとの往復文章で戻すUI操作結果を戻せるJSON、差分、プロンプトとして再投入しやすい。

FAQ：よくある疑問

記事中のFAQを、実務判断しやすい形に圧縮しました。

HTMLはトークン効率が悪くないのか？

Markdownのほうが短いことは多い。ただし、読まれる可能性と表現力を考えると、総合的な成果はHTMLのほうが高くなりやすい、という主張です。

Markdownはもう使わないのか？

短いメモ、README、簡単な手順書にはMarkdownが便利です。長い仕様書、比較検討、図解、操作が必要な成果物はHTMLが向きます。

計画書の代わりになるのか？

単一の計画書ではなく、探索用HTML、実装計画HTML、検証用HTMLのように、段階ごとに複数の参照ファイルを作るイメージです。

一番の価値は何か？

AIに作業を任せても、人間が判断の輪の中に残れること。HTMLは、AIの選択を見える形にし、人間が理解・修正・再指示しやすくします。

実践ポイント

この考え方を自分の作業に取り入れるなら、まずここからです。

① 読者を指定

「誰が」「何分で」「何を判断するか」を最初に書く。

② UI部品を指定

表、カード、サイドバー、タブ、検索、折りたたみを明示。

③ 図解を要求

データフロー、処理順、関係性はSVGやカードで表す。

④ 戻せる形にする

コピー用のJSON、Markdown、プロンプト、差分を出せるようにする。
